# Блог «МЕРИДИАН»

Статический блог на Astro с редактором на странице `/admin`. Материалы хранятся
markdown-файлами в вашем репозитории, сайт быстрый, обслуживать сервер не нужно.

Все нужные сервисы бесплатны: **GitHub** (хранит код и тексты), **Netlify**
(показывает сайт в интернете), **DecapBridge** (вход в редактор).

---

## Часть 1. Запуск — один раз, 30–60 минут

### Шаг 1. Положить код на GitHub
1. Заведите аккаунт на github.com.
2. Создайте новый репозиторий (New repository), назовите как угодно, оставьте Public, нажмите Create.
3. На странице репозитория: **Add file → Upload files**. Распакуйте этот архив и перетащите в окно загрузки **все файлы и папки, кроме `node_modules` и `dist`** (их в архиве и нет). Внизу нажмите **Commit changes**.

### Шаг 2. Показать сайт через Netlify
1. Заведите аккаунт на netlify.com (можно входом через GitHub).
2. **Add new site → Import an existing project → GitHub**, выберите ваш репозиторий.
3. Netlify сам подставит настройки из `netlify.toml` (команда `npm run build`, папка `dist`). Нажмите **Deploy**.
4. Через минуту-две сайт открывается по адресу вида `https://что-то.netlify.app`. В разделе **Site configuration → Change site name** можно задать понятное имя.

### Шаг 3. Включить вход в редактор (DecapBridge)
Netlify отключил свой встроенный логин, поэтому вход настраивается через DecapBridge.
1. Зайдите на decapbridge.com, создайте аккаунт (бесплатно).
2. **Create site**, привяжите ваш GitHub-репозиторий. Панель покажет готовый блок `backend` с вашими данными.
3. Откройте в репозитории файл `public/admin/config.yml`, найдите блок `backend` (вверху, со строками `REPLACE`) и **замените его на блок из DecapBridge**. Сохраните (Commit changes).
4. Там же в DecapBridge задайте себе пароль / пригласите свой e-mail. Если панель попросит добавить отдельный `<script>` — вставьте его в `public/admin/index.html` по их подсказке.
5. Готово: заходите на `ваш-сайт.netlify.app/admin` и логиньтесь.

### Шаг 4. Привести адреса в порядок (5 минут)
В двух местах стоит заглушка `https://example.netlify.app` — замените на реальный адрес сайта:
- `astro.config.mjs` — переменная `SITE` (нужно для RSS, карты сайта и поиска);
- `public/robots.txt` — строка `Sitemap:`.

Свой домен (если есть) подключается в Netlify: **Domain management → Add a domain**. Это можно сделать в любой момент позже.

## Часть 2. Как пользоваться каждый день

Всё происходит на `ваш-сайт/admin`:

- **Новый материал** — раздел «Материалы» → создать запись. Заполните заголовок, краткое описание (видно в ленте), рубрику, дату, при желании обложку — и текст. Нажмите публикацию. Сайт сам пересоберётся за 1–2 минуты, и материал появится на главной.
- **Редактировать / удалить** — там же, в списке материалов.
- **Рубрики** — отдельно заводить не нужно: впишите название рубрики в поле материала. Новое название автоматически станет новой рубрикой и фильтром на главной.
- **Картинки** — кнопка загрузки в поле «Обложка» или внутри текста; файлы сохраняются в `public/uploads`.
- **Название и подзаголовок сайта** — раздел «Настройки сайта».

Адрес материала формируется автоматически: русский заголовок превращается в
латинский (например «Каспий как перекрёсток» → `/posts/kaspiy-kak-perekrestok`).

## Проверить на компьютере (необязательно)
Нужен установленный Node.js (версия 18+).
```
npm install
npm run dev
```
Откроется `http://localhost:4321`. Чтобы редактор `/admin` работал локально, в
`config.yml` временно раскомментируйте `local_backend: true` и в отдельном окне
запустите `npx decap-server` (перед публикацией верните комментарий обратно).

---

## Где что лежит
- `src/content/posts/` — материалы (markdown). Три файла-образца можно удалить.
- `src/data/site.json` — название и описание сайта.
- `src/pages/` — страницы (главная, материал, рубрика, RSS).
- `src/styles/global.css` — весь дизайн.
- `public/admin/` — редактор (Decap CMS).
- `public/uploads/` — загруженные картинки.

## Если что-то не так
- **На `/admin` ошибка входа** — обычно блок `backend` в `config.yml` ещё не заменён на данные из DecapBridge (Шаг 3).
- **Материал не появился** — подождите 1–2 минуты (Netlify пересобирает) и обновите страницу; проверьте, что не включён «Черновик».
- **Сборка на Netlify упала** — откройте лог деплоя; чаще всего причина в опечатке в `config.yml` или `site.json` (нарушенный отступ/кавычки).
